# Contributing to NexusAgent

Thank you for your interest in contributing to NexusAgent! We welcome help from the community to make this project better for everyone.

## How to Contribute

### 1. Find an Issue
Check our [GitHub Issues](https://github.com/NexusAgent/nexusagent/issues) for open issues, or create a new one if you've found a bug or have a feature request.

### 2. Development Environment Setup
Follow our [Local Development Guide](docs/guides/local_development.md) to set up your environment.

### 3. Coding Standards
- **Style:** We follow the Google Python Style Guide. Use `ruff` for linting and formatting.
- **Type Checking:** We use `mypy` for strict type checking.
- **Testing:** All contributions must include tests. We use `pytest` for unit, integration, and E2E testing.

### 4. Pull Request Process
- **Branching:** Create a new branch for your work (e.g., `feature/my-new-feature` or `bugfix/fix-issue-123`).
- **Commits:** Use clear, descriptive, and imperative commit messages (e.g., `feat: Add new telemetry emitter`).
- **PRs:** Submit a Pull Request against the `main` branch. Each PR must include:
    - A clear description of the changes.
    - Links to related issues.
    - Passes all automated CI checks (linting, tests, build).
    - Has at least one approving review from a maintainer.

## Code of Conduct
Please be respectful and professional in all interactions. We aim to maintain a welcoming and inclusive environment for everyone.
