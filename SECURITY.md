# Security Policy

## Reporting a Vulnerability

We take the security of our software seriously. If you discover a security vulnerability within this project, please send an email to [security@nexusagent.example.com](mailto:security@nexusagent.example.com). All security vulnerabilities will be addressed promptly.

Please include the following information in your report:
- A description of the vulnerability
- Steps to reproduce the issue
- Any relevant logs or screenshots
- The version of the software you are using
- Any potential impact or exploitation scenarios

## Disclosure Policy

Upon receiving a vulnerability report, we will:
1. Acknowledge receipt of the report within 48 hours.
2. Investigate the issue and determine its validity and severity.
3. Keep the reporter informed of the progress of our investigation.
4. Once the issue is resolved, we will release a patch or update as soon as possible.
5. We will credit the reporter in our release notes (unless they wish to remain anonymous).

We ask that reporters:
- Give us reasonable time to resolve the issue before disclosing it publicly.
- Make a good faith effort to avoid privacy violations, destruction of data, and interruption or degradation of service during testing.
- Not exploit the vulnerability for any purpose beyond what is necessary to confirm its existence.

Thank you for helping to keep this project secure.
