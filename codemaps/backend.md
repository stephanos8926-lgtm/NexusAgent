## API Routes (FastAPI)
| Route | Method | Description |
|-------|--------|-------------|
| `/tasks` | POST | Submit a new task to the orchestrator |
| `/tasks/{id}/status` | GET | Poll for current task status (Pending/Processing/etc) |
| `/tasks/{id}/result` | GET | Retrieve the final agent result |
| `/health` | GET | Check NATS and Database connectivity |

## Core Services
- **NATSBus**: Wraps `nats-py` to handle JetStream KV operations and subject publishing/subscription.
- **NexusWorker**: The execution engine that listens for tasks, updates the DB state, and calls the agent logic.
- **Database Repository**: An `aiosqlite` wrapper for persisting `Task` and `Result` entities.

## Background Process Flow
1. **Worker Startup**: Started via FastAPI `lifespan` as an `asyncio` task.
2. **Subscription**: Listens on `tasks.submit`.
3. **Execution**: 
    - Mark task as `PROCESSING` in DB.
    - Run `agent.run_agent_task`.
    - Store result in NATS KV (for fast SDK retrieval).
    - Archive result in SQLite (for permanent record).
    - Mark task as `COMPLETED`/`FAILED` in DB.

## External Integrations
- **NATS Server**: Core messaging infrastructure.
- **SQLite**: Local state store.
- **LLM Providers**: Integrated via the `Agent` module.
