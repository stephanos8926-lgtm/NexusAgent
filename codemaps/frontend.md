## Component Hierarchy

### TUI (Textual)
- **NexusApp**
    - **Header** (System info)
    - **Log** (Streaming system output)
    - **Input** (Task definition entry)
    - **ErrorModal** (Blocking error feedback)
    - **Footer** (Keyboard shortcuts)

### Web UI (Gradio)
- **Blocks Layout**
    - **Control Center Header**
    - **Task Input Section**
        - Textbox (Task definition)
        - Submit Button
    - **Status Sidebar**
        - SDK Status Indicator
    - **Output Log**
        - System Result TextArea

## State Management
- **NexusSDK**: The sole source of truth for state synchronization. Clients call `sdk.submit_task()` and `sdk.get_result()`.
- **Local State**: Minimal local state in TUI/WebUI, primarily for UI responsiveness.

## Styling
- **TUI**: CSS-based terminal styling (dark theme, green-on-black logs).
- **Web UI**: Custom CSS for an "Industrial" look (Charcoal background, Orange-Red accents).
